Abstract of Findings

The main issue I found was SQL injection, which is a technique that attackers use to gain unauthorized access to a web application database by adding a string of malicious code to a database query.

Issues Found 1:

  • Issue: Unrestricted usernames
  • Location: Under app.post and app.get
  • Severity of issue: High. By not restriction user input, it allows user to send anything they want through the request and cause SQL injection

Issues Found 2:

  • Issue: Mixing userinput with code together
  • Location: inside theQuery itself
  • Severity of issue: High. SQL Injection. By mixing the user input and the code, we are giving any access to data that they should not have access to
  • Resolution: to prevent this issue, we can use prepare statement. Separete sql statement with user input
