Introduction

I am a security consultant, I was hired by a ride-hailing service called NotUber to document and resolve the security and privacy issues of their website

Methodology

My methodology pen testing the system is utilizing the knowledge I learn in class and apply them by reading through the source code

Abstract of Findings

The main issues I found are SQL injection and Cross-site scripting attack.

A SQL injection which is a technique that attackers use to gain unauthorized access to a web application database by adding a string of malicious code to a database query

Cross site scripting (XSS) is an attack in which an attacker injects malicious executable scripts into the code of a trusted application or website. Attackers often initiate an XSS attack by sending a malicious link to a user and enticing the user to click it

Issues Found 1:

  • Issue: Unrestricted usernames
  • Location: Under app.post and app.get
  • Severity of issue: High. By not restriction user input, it allows user to send anything they want through the request and cause SQL injection

Issues Found 2:

  • Issue: Mixing userinput with code together
  • Location: inside theQuery itself
  • Severity of issue: High. Data breach. By mixing the user input and the code, we are giving any access to data or system that they should not have access to anyone. SQL injection attacks allow attackers to spoof identity, tamper with existing data, cause repudiation issues such as voiding transactions or changing balances, allow the complete disclosure of all data on the system, destroy the data or make it otherwise unavailable, and become administrators of the database server.
  • Resolution: to prevent this issue, we can use prepare statement. Separete sql statement with user input

Issues Found 3:

  • Issue: Cross-site scripting (XSS)
  • Location: inside the response header 'response.header("Access-Control-Allow-Origin", "*");'
  • Severity of issue: High. Severe XSS attacks can cause disclosure of the user's session cookie, allowing an attacker to hijack the user's session and take over the account
  • Resolution: To prevent XSS attacks, application must validate all the input data, make sure that only the allowlisted data is allowed, and ensure that all variable output in a page is encoded before it is returned to the user

Conclusion

There are several ways to prevent hackers from penetrating the site or personal account:

  1. Watch out for SQL injection
  2. Protect against XSS attacks
  3. Beware of error messages: Be careful with how much information you give away in your error messages
  4. Validate on both sides: Validation should always be done both on the browser and server side
  5. Use HTTPS
